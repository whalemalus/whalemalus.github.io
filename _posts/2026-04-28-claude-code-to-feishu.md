# 把 Claude Code 接入飞书：从终端到企业 IM 的 AI 编码助手\n\n> **摘要**：Claude Code 在终端里跑得很好，但团队协作时总不能每个人都 SSH 到服务器。这篇文章记录了如何用一个 Python 文件 + WebSocket，把 Claude Code 桥接到飞书，让团队在 IM 里直接使用 AI 编码能力。\n>\n> **关键词**：`Claude Code` `飞书机器人` `WebSocket` `AI Agent` `企业 IM`\n\n---\n\n## 楔子\n\n周五下午，同事在飞书群里问：\n\n> "这个 Python 报错怎么解决？"附了一张截图。\n\n我切到终端，打开 Claude Code，把报错贴进去，等它分析完，复制结果，切回飞书，粘贴发送。\n\n三分钟后，另一个同事又问了一个类似的问题。\n\n我重复了一遍同样的操作。\n\n那一刻我在想：**Claude Code 已经能处理这些问题了，为什么我还要当中间人？**\n\n---\n\n## 引言\n\n这篇文章要做的事很简单：**把 Claude Code 从你的终端搬到飞书里。**\n\n完成后，团队成员在飞书群里 @机器人 发一段报错，就能实时看到 Claude Code 的分析过程——读了哪个文件、跑了什么命令、最终结论是什么。\n\n不需要公网 IP，不需要域名，不需要 API 网关。一个 Python 文件 + Systemd，5 分钟部署完成。\n\n技术栈：Claude Code CLI、飞书开放平台 WebSocket API、Python lark-oapi。\n\n---\n\n## 1. 全景地图：系统架构\n\n```\n┌─────────────┐     WebSocket       ┌──────────────┐     CLI (subprocess)     ┌─────────────┐\n│   飞书用户   │ ←────────────────→ │  Python Bot   │ ──────────────────────→ │ Claude Code │\n│  (手机/PC)   │    飞书长连接       │  (服务器运行)  │                          │    CLI      │\n└─────────────┘                     └──────────────┘                          └─────────────┘\n                                          │\n                                          │ 实时反馈\n                                          ▼\n                                    ⚙️ 步骤1: `cat auth.py`\n                                    📝 步骤2: 写入 test_auth.py\n                                    ✅ 完成\n```\n\n**关键设计决策：Bot 不做任何 AI 处理，只做桥接。**\n\n为什么要这样？因为 Claude Code 本身已经是一个完整的 AI Agent——它能读文件、写代码、跑命令、搜索代码库。Bot 只需要把飞书的消息传给它，把它的输出传回飞书。\n\n这意味着：Claude Code 终端里能做的事，飞书里都能做。\n\n---\n\n## 2. 核心概念\n\n### 2.1 为什么用 WebSocket 而不是 Webhook？\n\n飞书机器人有两种接收消息的方式：\n\n| 方式 | 需要公网 IP | 延迟 | 部署难度 |\n|------|------------|------|---------|\n| Webhook（HTTP 回调） | ✅ 需要 | 低 | 高（要配域名、SSL、反向代理） |\n| WebSocket（长连接） | ❌ 不需要 | 低 | 低（一个 Python 进程） |\n\nWebSocket 方案的优势是**不需要公网 IP**。Bot 主动连接飞书服务器，建立长连接后双向通信。对于部署在内网或没有域名的服务器来说，这是唯一的选择。\n\n飞书的 WebSocket API 通过 `lark_oapi.ws.Client` 实现，底层是 `wss://msg-frontier.feishu.cn/ws/v2`。\n\n### 2.2 Claude Code 的 stream-json 输出\n\nClaude Code CLI 支持 `--output-format stream-json` 参数，可以实时输出 JSON 流：\n\n```bash\nclaude -p "分析这段代码" --output-format stream-json --verbose\n```\n\n每行是一个 JSON 对象，包含：\n- `type: "assistant"` — AI 的思考和工具调用\n- `type: "result"` — 最终结果\n\nBot 解析这些 JSON 事件，提取工具调用信息，实时推送到飞书：\n\n```json\n{"type":"assistant","message":{"content":[{"type":"tool_use","name":"Bash","input":{"command":"cat auth.py"}}]}}\n{"type":"assistant","message":{"content":[{"type":"tool_use","name":"Write","input":{"file_path":"/tmp/test.py"}}]}}\n{"type":"result","result":"分析完成，问题在于..."}\n```\n\n### 2.3 会话隔离\n\n每个飞书聊天（私聊或群聊）有独立的 `chat_id`。Bot 根据 `chat_id` 创建独立的工作目录：\n\n```\n/home/claude-user/Claude Projects/\n├── chat_12345678/    ← 用户A的工作目录\n├── chat_87654321/    ← 用户B的工作目录\n└── chat_11223344/    ← 群聊C的工作目录\n```\n\n这样不同用户的文件操作互不干扰。\n\n---\n\n## 3. 实战指南：5 分钟部署\n\n### 3.1 前置条件\n\n- Claude Code 已安装并配置好（`claude --version` 能正常输出）\n- 飞书企业自建应用（[创建指南](https://open.feishu.cn/app)）\n- Python 3.10+\n\n### 3.2 创建飞书应用\n\n1. 登录 [飞书开发者后台](https://open.feishu.cn/app)\n2. 创建企业自建应用\n3. 记录 `App ID` 和 `App Secret`\n\n### 3.3 一键部署\n\n```bash\n# 克隆项目\ngit clone https://github.com/whalemalus/claude-code-to-feishu.git\ncd claude-code-to-feishu\n\n# 配置环境变量\ncp .env.example .env\nnano .env\n# 填入 FEISHU_APP_ID 和 FEISHU_APP_SECRET\n\n# 一键部署\nchmod +x setup.sh\n./setup.sh\n```\n\n`setup.sh` 会自动：\n1. 安装系统依赖（python3、pip）\n2. 安装 Python 包（lark-oapi）\n3. 创建 `claude-user`（Claude Code 禁止以 root 运行）\n4. 注册 Systemd 服务\n\n### 3.4 飞书后台配置\n\n回到飞书开发者后台：\n\n1. **开启机器人能力**：应用功能 → 机器人 → 启用\n2. **添加权限**：\n   - `im:message` — 发送消息\n   - `im:message:send_as_bot` — 以机器人身份发送\n   - `im:message.p2p_msg:readonly` — 接收私聊消息\n3. **订阅事件**：事件与回调 → 事件配置 → 添加 `im.message.receive_v1`\n4. **发布应用**：版本管理与发布 → 创建版本 → 发布\n\n### 3.5 开始使用\n\n在飞书中搜索你的机器人名称，直接发消息：\n\n```\n你: 帮我写一个快速排序算法\n\nBot: 🤔 步骤1: `cat > quicksort.py << 'EOF'...`\nBot: 📝 步骤2: 写入 quicksort.py\nBot: ⚙️ 步骤3: `python3 quicksort.py`\nBot: ✅ 完成\n     📋 cat → Write → Bash\n     \n     排序结果：[1, 2, 3, 5, 8]\n```\n\n---\n\n## 4. 技术实现详解\n\n### 4.1 消息处理流程\n\n```python\ndef handle_message(data):\n    msg = data.event.message\n    text = extract_text(msg.content)\n    chat_id = msg.chat_id\n    \n    # 特殊命令\n    if text == "/clear":\n        shutil.rmtree(get_work_dir(chat_id))\n        send_feishu(chat_id, "🔄 已清除")\n        return\n    \n    # 在新线程中调用 Claude Code\n    threading.Thread(\n        target=call_claude_code,\n        args=(chat_id, text, msg.message_id)\n    ).start()\n```\n\n为什么用新线程？因为 Claude Code 处理任务可能需要几十秒到几分钟。如果在主线程阻塞，会丢失其他用户的消息。\n\n### 4.2 Claude Code 调用与实时反馈\n\n```python\ndef call_claude_code(chat_id, user_message, user_msg_id):\n    # 构建命令\n    shell_cmd = (\n        f"cd '{work_dir}' && "\n        f"claude -p '{escaped_msg}' "\n        f"--dangerously-skip-permissions "\n        f"--output-format stream-json "\n        f"--verbose "\n        f"--max-turns 15"\n    )\n    \n    # 添加思考表情 🤔\n    thinking_reaction = add_reaction(user_msg_id, "THINKING")\n    \n    # 流式读取输出\n    process = subprocess.Popen(\n        ["sudo", "-u", CLAUDE_USER, "bash", "-c", shell_cmd],\n        stdout=subprocess.PIPE, text=True, bufsize=1\n    )\n    \n    for line in process.stdout:\n        event = json.loads(line)\n        \n        if event["type"] == "assistant":\n            for block in event["message"]["content"]:\n                if block["type"] == "tool_use":\n                    name = block["name"]\n                    # 实时推送到飞书\n                    if name == "Bash":\n                        send_feishu(chat_id, f"⚙️ `{block['input']['command'][:80]}`")\n                    elif name == "Write":\n                        send_feishu(chat_id, f"📝 写入 `{block['input']['file_path']}`")\n                    elif name == "Read":\n                        send_feishu(chat_id, f"📖 读取 `{block['input']['file_path']}`")\n    \n    # 删除思考表情，添加完成表情 ✅\n    remove_reaction(user_msg_id, thinking_reaction)\n    add_reaction(user_msg_id, "DONE")\n    send_feishu(chat_id, f"✅ 完成\\n\\n{final_text}")\n```\n\n### 4.3 消息去重\n\n飞书的 WebSocket 可能会重复推送同一条消息。用一个简单的内存字典去重：\n\n```python\nprocessed_messages = {}\n\ndef is_duplicate(message_id):\n    now = time.time()\n    # 清理过期记录\n    expired = [k for k, v in processed_messages.items() if now - v > 60]\n    for k in expired:\n        del processed_messages[k]\n    \n    if message_id in processed_messages:\n        return True\n    processed_messages[message_id] = now\n    return False\n```\n\n---\n\n## 5. 踩坑记录\n\n### 坑 1：Claude Code 禁止以 root 运行\n\n```\nError: --dangerously-skip-permissions cannot be used with root/sudo privileges\n```\n\n**原因**：Claude Code 有安全限制，root 用户不能跳过权限检查。\n\n**解决**：创建专用的 `claude-user`，用 `sudo -u claude-user` 执行 Claude Code：\n\n```bash\nuseradd -m -s /bin/bash claude-user\n```\n\n### 坑 2：Python 包名是 lark-oapi 不是 lark_oapi\n\n```bash\npip3 install lark-oapi  # ✅ 正确（包名用连字符）\nimport lark_oapi as lark  # ✅ 导入用下划线\n```\n\nPython 的包名和导入名不一致，容易搞混。\n\n### 坑 3：飞书事件订阅需要先保存再发布\n\n配置事件订阅后，必须：\n1. 先保存配置\n2. 创建新版本\n3. 发布版本\n\n如果跳过"创建版本"直接测试，事件不会推送。\n\n### 坑 4：Antropic API Gateway（AxonHub）的可选集成\n\n如果你不想直接用 Anthropic 官方 API（比如想用第三方模型或想省钱），可以在 Claude Code 前面加一个 API 网关：\n\n```env\n# .env\nANTHROPIC_API_KEY=your-gateway-key\nANTHROPIC_BASE_URL=http://your-server:8090/anthropic\n```\n\nBot 的 `call_claude_code` 会把这些环境变量注入到 Claude Code 的执行环境中。这样 Claude Code 实际调用的是你的网关，网关再转发到任意模型。\n\n---\n\n## 6. 总结与展望\n\n### 核心收获\n\n1. **Bot 只做桥接，不做 AI**。让 Claude Code 做它擅长的事，Bot 只负责消息转发。\n2. **WebSocket 优于 Webhook**。不需要公网 IP，部署简单，延迟低。\n3. **stream-json 是关键**。没有它，用户只能看到最终结果；有了它，用户能实时看到每一步操作。\n4. **会话隔离很重要**。不同用户的工作目录必须独立，否则文件操作会互相干扰。\n\n### 可以做什么\n\n- 团队代码审查：在群里贴代码，机器人自动审查\n- 快速原型：描述需求，机器人直接生成可运行的代码\n- Bug 诊断：贴报错信息，机器人自动分析原因\n- 文档生成：指定代码目录，机器人生成 API
