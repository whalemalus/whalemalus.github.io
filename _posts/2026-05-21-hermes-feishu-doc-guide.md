# Hermes Agent 接入飞书云文档完全指南：从零到读写文档\n\n> **摘要**：手把手教你让 AI 智能体（Hermes Agent）通过飞书开放平台 API 读取、搜索、编辑飞书云文档。两种方案（官方 CLI + 原生 API），附完整踩坑记录。\n>\n> **关键词**：`Hermes Agent` `飞书开放平台` `lark-cli` `云文档API` `AI自动化`\n\n---\n\n## 楔子\n\n凌晨两点，你还在手动把飞书文档里的会议纪要复制到 AI 助手的对话框里。复制、粘贴、提问、复制回答、切回飞书、粘贴——来回七八次，脖子已经僵了。\n\n你心想：能不能让 AI 直接读我的飞书文档？甚至帮我把总结写回飞书？\n\n答案是：可以。而且只需要 30 分钟配置。\n\n---\n\n## 引言\n\n飞书（Lark）是国内最主流的企业协作平台之一，大量团队的知识资产都沉淀在飞书云文档中。Hermes Agent 是一个开源的 AI 智能体框架，支持工具调用、定时任务、多平台接入。\n\n将两者打通后，你可以：\n- **让 AI 读取飞书文档**：发一个链接，AI 自动获取全文内容\n- **让 AI 搜索飞书文档**：用自然语言在知识库中检索\n- **让 AI 编辑飞书文档**：自动生成总结、更新文档内容\n- **自动化工作流**：定时扫描文档 → AI 分析 → 写回结果\n\n本文介绍两种接入方案，从简单的官方 CLI 到灵活的原生 API。\n\n---\n\n## 📖 目录\n\n1. 搜索文档                        │\n│  ✅ 编辑/追加内容   ✅ 创建新文档                      │\n│  ✅ 发送飞书消息    ✅ 管理群组                        │\n│  ✅ 日历/任务操作   ✅ 多维表格操作 文档/消息/日历/表格 | 仅文档读取 |\n| 认证方式 | OAuth 一次性登录 | App Token 定期刷新 |\n| 推荐场景 | 日常使用 | lark-cli 不可用时 |\n\n---\n\n## 方案一：lark-cli 官方工具（推荐）\n\n### 第一步：安装 lark-cli\n\n```bash\nnpm install -g @larksuite/cli\nlark-cli --version  # 验证安装成功\n```\n\n需要 Node.js v18+。\n\n### 第二步：绑定到你的 AI 智能体\n\n```bash\nlark-cli config bind --source hermes --identity user-default\n```\n\n这一步告诉 lark-cli："我在 Hermes Agent 环境中运行，请用用户身份操作。"\n\n**身份选择：**\n- `user-default`：以你的个人身份操作，能访问你的私人文档、日历、邮件。**读取个人文档必须选这个**。\n- `bot-only`：仅用机器人身份，安全但无法访问个人资源。\n\n### 第三步：OAuth 登录（一次性）\n\n```bash\n# 第一轮：获取授权链接\nlark-cli auth login --recommend --no-wait --json\n```\n\n输出中有一个 `verification_url`，把它发给用户（或者自己打开），在浏览器中完成授权。\n\n```bash\n# 用户授权后，用返回的 device_code 完成登录\nlark-cli auth login --device-code "<device_code>" --json\n```\n\n**关键：OAuth 必须分批授权，不能一次请求所有权限。** 第一轮用 `--recommend` 获取基础读取权限。如果后续需要搜索或发消息，再追加：\n\n```bash\n# 追加搜索权限\nlark-cli auth login --scope "search:docs:read" --no-wait --json\n\n# 追加发消息权限\nlark-cli auth login --scope "im:message.send_as_user" --no-wait --json\n```\n\n### 第四步：验证登录状态\n\n```bash\nlark-cli auth status\n```\n\n显示已登录的用户信息即表示成功。\n\n---\n\n## 方案二：飞书开放 API（备用）\n\n当 lark-cli 不可用时，可以直接调用飞书 Open API。\n\n### 第一步：创建飞书应用\n\n1. 打开 [飞书开放平台](https://open.feishu.cn/) → 创建企业自建应用\n2. 在"权限管理"中申请以下权限：\n   - `docx:document:readonly` — 读取文档\n   - `wiki:wiki:readonly` — 读取知识库\n   - `docx:document` — 编辑文档（如需写入）\n3. 获取 `App ID` 和 `App Secret`，保存到环境变量：\n\n```bash\n# 写入 ~/.hermes/.env\necho "FEISHU_APP_ID=cli_xxxxx" >> ~/.hermes/.env\necho "FEISHU_APP_SECRET=xxxxx" >> ~/.hermes/.env\n```\n\n### 第二步：获取 Tenant Access Token\n\n```bash\nsource ~/.hermes/.env\nTOKEN=$(curl -s -X POST "https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal" \\\n  -H "Content-Type: application/json" \\\n  -d "{\\"app_id\\":\\"${FEISHU_APP_ID}\\",\\"app_secret\\":\\"${FEISHU_APP_SECRET}\\"}" \\\n  | python3 -c "import json,sys; print(json.load(sys.stdin)['tenant_access_token'])")\necho "Token: ${TOKEN:0:20}..."\n```\n\nToken 有效期约 2 小时，长时间操作需要刷新。\n\n### 第三步：提取文档 Token\n\n从飞书文档 URL 中提取 token：\n- Wiki 格式：`https://xxx.feishu.cn/wiki/{DOC_TOKEN}`\n- Docx 格式：`https://xxx.feishu.cn/docx/{DOC_TOKEN}`\n\n### 第四步：获取文档内容（分页）\n\n```bash\n# 获取文档元信息\ncurl -s "https://open.feishu.cn/open-apis/docx/v1/documents/${DOC_TOKEN}" \\\n  -H "Authorization: Bearer ${TOKEN}"\n\n# 获取文档块（每页最多 500 个 block）\ncurl -s "https://open.feishu.cn/open-apis/docx/v1/documents/${DOC_TOKEN}/blocks?page_size=500" \\\n  -H "Authorization: Bearer ${TOKEN}"\n```\n\n如果返回 `data.has_more: true`，用 `data.page_token` 参数获取下一页。\n\n---\n\n## 实战：读取飞书文档\n\n### 用 lark-cli 读取（最简单）\n\n```bash\n# 直接传 URL\nlark-cli docs +fetch --doc "https://xxx.feishu.cn/wiki/ABC123" --format pretty\n\n# 传文档 token\nlark-cli docs +fetch --doc "ABC123" --api-version v2 --format pretty\n```\n\n输出就是文档的全文内容，可以直接交给 AI 分析。\n\n### 搜索飞书文档\n\n```bash\nlark-cli docs +search --query "项目计划" --page-size 5 --format pretty\n```\n\n### 读取知识库（Wiki）\n\n```bash\n# 列出知识库空间\nlark-cli wiki +space-list --format pretty\n\n# 获取 wiki 节点信息\nlark-cli wiki +node-get --token "https://feishu.cn/wiki/ABC123" --format json\n```\n\n---\n\n## 实战：编辑飞书文档\n\n### 追加内容到文档末尾\n\n```bash\n# 先写内容到临时文件\ncat > /tmp/ai-summary.md << 'EOF'\n## AI 自动生成摘要\n\n本文主要讨论了以下要点：\n\n1. **核心结论**：XXX\n2. **行动计划**：XXX\n3. **风险提示**：XXX\n\n> 此摘要由 Hermes Agent 自动生成\nEOF\n\n# 追加到文档\nlark-cli docs +update --doc "ABC123" --mode append --markdown @/tmp/ai-summary.md\n```\n\n### 在指定位置插入内容\n\n```bash\n# 在某个章节标题后面插入\nlark-cli docs +update --doc "ABC123" --mode insert_after \\\n  --selection-by-title "## 会议记录" \\\n  --markdown @/tmp/new-content.md\n\n# 在模糊匹配的文本后面插入\nlark-cli docs +update --doc "ABC123" --mode insert_after \\\n  --selection-with-ellipsis "讨论了...最终决定" \\\n  --markdown @/tmp/follow-up.md\n```\n\n### 覆盖整个文档\n\n```bash\nlark-cli docs +update --doc "ABC123" --mode overwrite --markdown @/tmp/full-content.md\n```\n\n### 创建新文档\n\n```bash\ncd /path/to/file && lark-cli docs +create \\\n  --title "AI 自动生成报告" \\\n  --markdown @report.md \\\n  --wiki-space my_library \\\n  --as user \\\n  --jq '.doc_url'\n```\n\n---\n\n## 踩坑记录\n\n### 坑 1：OAuth 权限必须分批授权\n\n**现象**：一次性请求所有权限（读文档 + 搜索 + 发消息 + 日历），OAuth 页面直接报错。\n\n**原因**：飞书 OAuth 对单次请求的 scope 数量和组合有限制。\n\n**解决**：分 2-3 轮授权。第一轮用 `--recommend`，后续按需追加。\n\n### 坑 2：lark-cli 必须先绑定环境\n\n**现象**：直接运行 `lark-cli auth login` 报错 "hermes context detected but lark-cli is not bound to it"。\n\n**解决**：先执行 `lark-cli config bind --source hermes --identity user-default`。\n\n### 坑 3：Wiki 页面需要额外处理\n\n**现象**：用 wiki URL 直接读取文档，返回内容为空或报权限错误。\n\n**原因**：Wiki 的 URL 中的 token 是 `node_token`，不是文档的 `obj_token`。\n\n**解决**：\n```bash\n# 先获取 obj_token\nlark-cli wiki +node-get --token "https://feishu.cn/wiki/NODE_TOKEN" --format json\n# 用返回的 obj_token 读取文档\nlark-cli docs +fetch --doc "<obj_token>" --format pretty\n```\n\n### 坑 4：`docs +update` 必须用 v1 API\n\n**现象**：`lark-cli docs +update --api-version v2` 报错 "command is required"。\n\n**解决**：`docs +fetch` 用 v2，`docs +update` 用 v1（默认）。\n\n### 坑 5：`--markdown` 参数的文件路径\n\n**现象**：`lark-cli docs +create --markdown /absolute/path/file.md` 报错 "must be a relative path"。\n\n**解决**：先 `cd` 到文件所在目录，然后用相对路径：`--markdown @file.md`。\n\n### 坑 6：内置 `feishu_doc_read` 工具的限制\n\n**现象**：在普通对话中调用 `feishu_doc_read(doc_token)` 报错 "Feishu client not available (not in a Feishu comment context)"。\n\n**原因**：Hermes 的内置飞书工具只在飞书评论上下文中可用。\n\n**解决**：改用 `lark-cli docs +fetch`，它在任何上下文中都能工作。\n\n### 坑 7：Token 过期\n\n**现象**：运行一段时间后 API 调用返回 401。\n\n**原因**：User Token 有效期约 2 小时，Refresh Token 有效期 7 天。\n\n**解决**：重新执行 `lark-cli auth login` 刷新授权。\n\n---\n\n## 总结与展望\n\n### 核心收获\n\n1. **lark-cli 是首选方案**：安装简单、功能全面、自动处理认证和分页\n2. **OAuth 分批授权是必须的**：不要试图一次申请所有权限\n3. **Wiki 页面需要 token 转换**：node_token ≠ obj_token\n4. **读写分离的权限设计**：只读用 `--recommend`，写入需要额外授权\n\n### 最佳实践\n\n- 生产环境用 `bot-only` 身份，开发调试用 `user-default`\n- 文档操作前先 `lark-cli auth status` 确认登录状态\n- 大文档分页处理，不要一次拉取所有内容\n- 编辑操作先在测试文档上验证\n\n### 延伸阅读\n\n- [飞书开放平台官方文档](https://open.feishu.cn/document/home/index)\n- [lark-cli GitHub 仓库](https://github.com/larksuite/cli)\n- [Hermes Agent
