# PageWise × PinchTab E2E 测试实践：Chrome Extension 的无浏览器自动化测试方案\n\n> 2026-05-16 | 技术实践\n\n## 背景\n\nChrome Extension 的 E2E 测试一直是个难题——依赖真实的 Chrome 浏览器环境、需要 `--load-extension` 启动参数、Service Worker 生命周期特殊……有没有办法用 PinchTab（headless Chrome 自动化工具）来测试？\n\n本文记录了 PageWise 项目使用 PinchTab 进行 E2E 测试的完整实践。\n\n## 方案分析\n\n### PinchTab 能做什么？\n\n| 测试类型 | 可行性 | 说明 |\n|---------|--------|------|\n| UI 渲染/布局 | ✅ | HTML/CSS 直接渲染 |\n| 表单交互 | ✅ | 输入框、下拉框、按钮 |\n| 主题/样式 | ✅ | CSS 切换、深色模式 |\n| Tab 切换 | ✅ | DOM 操作验证 |\n| Chrome Extension API | ❌ | 非扩展上下文 |\n| Service Worker | ❌ | 需要真实扩展安装 |\n| Content Script 注入 | ⚠️ | 需手动 JS 注入 |\n\n### 核心限制\n\nPinchTab 打开的页面是普通网页，不是扩展上下文。`chrome.storage`、`chrome.runtime`、`chrome.sidePanel` 等 API 不存在。\n\n### 方案A：Mock Chrome API + 独立页面测试\n\n**核心思路**：在页面加载前注入 Chrome API Mock，让 UI 代码可以脱离扩展独立运行。\n\n## 实现步骤\n\n### Step 1: 创建 Chrome Mock 注入脚本\n\n```javascript\n// tests/e2e/chrome-mock-inject.js\n(function() {\n  function createStore() {\n    const store = {};\n    return {\n      get: (keys, cb) => {\n        const result = {};\n        if(!keys) Object.assign(result, store);\n        else if(typeof keys === 'string') result[keys] = store[keys];\n        else if(Array.isArray(keys)) keys.forEach(k => result[k] = store[k]);\n        if(cb) cb(result);\n        return Promise.resolve(result);\n      },\n      set: (items, cb) => { Object.assign(store, items); if(cb) cb(); return Promise.resolve(); },\n      remove: (keys, cb) => { return Promise.resolve(); },\n      clear: (cb) => { return Promise.resolve(); }\n    };\n  }\n\n  window.chrome = {\n    storage: { local: createStore(), sync: createStore(), session: createStore() },\n    runtime: { \n      id: 'mock-ext', \n      sendMessage: (m,cb) => { if(cb) cb({ok:true}); return Promise.resolve({ok:true}); },\n      onMessage: { addListener:()=>{}, removeListener:()=>{} },\n      getURL: (p) => p \n    },\n    tabs: { query: (q,cb) => { if(cb) cb([{id:1,url:'test',active:true}]); return Promise.resolve([]); } },\n    sidePanel: { setOptions: () => Promise.resolve() }\n  };\n  window.__PINCHTAB_MOCK__ = true;\n})();\n```\n\n### Step 2: 用 PinchTab 注入 Mock 并创建 UI\n\n```bash\n# 1. 导航到任意页面\ndocker exec pinchtab pinchtab nav "https://example.com" --snap\n\n# 2. 注入 Chrome Mock\ndocker exec pinchtab pinchtab eval "\n(function() {\n  // ... mock 代码 ...\n  window.chrome = { storage: { local: createStore() }, ... };\n  return 'mock installed';\n})()\n"\n\n# 3. 创建 PageWise UI 结构\ndocker exec pinchtab pinchtab eval "\n(function() {\n  document.body.innerHTML = `\n    <div id='app'>\n      <nav role='tablist'>\n        <button class='tab active' data-tab='chat'>问答</button>\n        <button class='tab' data-tab='knowledge'>知识</button>\n        <button class='tab' data-tab='settings'>设置</button>\n      </nav>\n      <!-- 面板内容 -->\n    </div>\n  `;\n  // 绑定事件...\n  return 'UI created';\n})()\n"\n```\n\n### Step 3: 测试交互\n\n```bash\n# 测试 Tab 切换\ndocker exec pinchtab pinchtab eval "\n(function() {\n  document.querySelectorAll('.tab')[1].click();\n  return 'tab switched';\n})()\n"\n\n# 测试表单输入\ndocker exec pinchtab pinchtab eval "\n(function() {\n  document.getElementById('userInput').value = '什么是闭包?';\n  document.getElementById('btnSend').click();\n  return 'message sent';\n})()\n"\n\n# 测试 Storage 读写\ndocker exec pinchtab pinchtab eval "\n(function() {\n  chrome.storage.local.set({theme: 'dark'});\n  var result;\n  chrome.storage.local.get(['theme'], function(r) { result = r; });\n  return 'storage: ' + JSON.stringify(result);\n})()\n"\n```\n\n## 测试结果\n\n### ✅ 通过的测试\n\n| 测试项 | 结果 | 说明 |\n|--------|------|------|\n| Chrome Mock 注入 | ✅ | 所有 API 可用 |\n| UI 渲染 | ✅ | 8 个按钮正确渲染 |\n| Tab 切换 | ✅ | 面板显示/隐藏正确 |\n| 表单输入 | ✅ | textarea 值设置成功 |\n| Storage 读写 | ✅ | set/get 往返一致 |\n| 主题切换 | ✅ | 深色模式应用成功 |\n| 按钮点击事件 | ✅ | 事件处理器触发 |\n\n### 截图验证\n\n**问答面板**：欢迎界面、快捷按钮、输入框正常渲染\n\n**设置面板**：API Key 输入框、主题下拉框、保存按钮正常工作\n\n**深色主题**：背景色切换为 #1a1a2e，文字颜色适配\n\n## 局限性\n\n1. **无法测试真实 Chrome API 调用**：`chrome.storage` 的实际持久化、`chrome.runtime.sendMessage` 的跨上下文通信\n2. **无法测试 Service Worker**：MV3 的后台脚本生命周期\n3. **无法测试 Content Script 注入**：自动注入到目标网页的机制\n4. **UI 结构需要手动维护**：测试用的 HTML 结构与实际 sidebar.html 可能不同步\n\n## 最佳实践\n\n1. **Mock 要轻量**：只 mock 实际用到的 API，不要试图完整模拟 Chrome\n2. **测试关注交互**：Tab 切换、表单提交、状态变更，而非 API 实现细节\n3. **截图辅助验证**：PinchTab 的 screenshot 功能用于视觉回归测试\n4. **与单元测试互补**：E2E 测试 UI 交互，单元测试验证逻辑正确性\n\n## 总结\n\n方案A验证了 PinchTab 可以用于 Chrome Extension 的 UI 层 E2E 测试。虽然无法覆盖 Extension API 的真实调用，但对于 Tab 切换、表单交互、主题切换等用户可见的行为，PinchTab 提供了一种轻量级的自动化测试方案。\n\n核心价值：**不需要启动真实 Chrome + 加载扩展，就能验证 UI 交互逻辑是否正确。**\n
