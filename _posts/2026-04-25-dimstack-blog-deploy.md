Dockerfile 适配修改\n\n### ⚠️ 问题发现\n\n原始 Dockerfile 中使用了**中国镜像源**（清华镜像），这在国内服务器上构建速度很快，但在**海外服务器上可能不可用或极慢**：\n\n```dockerfile\n# 原始配置（中国镜像）\nRUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.tuna.tsinghua.edu.cn/g' /etc/apk/repositories\n```\n\n### 🔧 解决方案\n\n根据服务器所在地区选择合适的镜像源：\n\n**海外服务器**（使用默认镜像）：\n\n```dockerfile\n# 无需修改，或显式指定\nRUN sed -i 's/dl-cdn.alpinelinux.org/dl-cdn.alpinelinux.org/g' /etc/apk/repositories\n```\n\n实际上直接删除或注释掉该行即可：\n\n```dockerfile\n# 海外服务器 - 注释掉中国镜像\n# RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.tuna.tsinghua.edu.cn/g' /etc/apk/repositories\n```\n\n**国内服务器**（保持原样）：\n\n```dockerfile\n# 国内服务器 - 使用清华镜像加速\nRUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.tuna.tsinghua.edu.cn/g' /etc/apk/repositories\n```\n\n### 📝 修改后的完整 Dockerfile\n\n```dockerfile\nFROM eclipse-temurin:21-jre-alpine\n\nWORKDIR /dim_stack\n\n# 海外服务器注释掉下行，国内服务器保留\n# RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.tuna.tsinghua.edu.cn/g' /etc/apk/repositories\n\nCOPY target/*.jar app.jar\n\nEXPOSE 2222\n\nENTRYPOINT ["java", "-jar", "app.jar"]\n```\n\n---\n\n## 5. 编写 docker-compose.yml\n\n由于项目没有提供 `docker-compose.yml`，我们需要自行编写一个完整的编排文件，将应用、MySQL、Redis 三个服务统一管理。\n\n### 📝 完整配置\n\n```yaml\nversion: '3.8'\n\nservices:\n  # ==================== MySQL 数据库 ====================\n  mysql:\n    image: mysql:8.0\n    container_name: dimstack-mysql\n    restart: unless-stopped\n    environment:\n      MYSQL_ROOT_PASSWORD: DimStack@2026\n      MYSQL_DATABASE: dim_stack\n      MYSQL_CHARSET: utf8mb4\n      MYSQL_COLLATION: utf8mb4_unicode_ci\n    ports:\n      - "127.0.0.1:3307:3306"\n    volumes:\n      - ./mysql-data:/var/lib/mysql\n    healthcheck:\n      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]\n      interval: 10s\n      timeout: 5s\n      retries: 10\n      start_period: 30s\n    networks:\n      - dimstack-net\n\n  # ==================== Redis 缓存 ====================\n  redis:\n    image: redis:7-alpine\n    container_name: dimstack-redis\n    restart: unless-stopped\n    command: redis-server --appendonly yes\n    ports:\n      - "127.0.0.1:6380:6379"\n    volumes:\n      - ./redis-data:/data\n    healthcheck:\n      test: ["CMD", "redis-cli", "ping"]\n      interval: 10s\n      timeout: 5s\n      retries: 5\n    networks:\n      - dimstack-net\n\n  # ==================== Dim Stack 应用 ====================\n  dimstack:\n    build:\n      context: .\n      dockerfile: Dockerfile\n    container_name: dimstack-app\n    restart: unless-stopped\n    depends_on:\n      mysql:\n        condition: service_healthy\n      redis:\n        condition: service_healthy\n    ports:\n      - "2222:2222"\n    volumes:\n      - ./config:/dim_stack/config\n      - ./upload:/dim_stack/upload\n      - ./logs:/dim_stack/logs\n      - ./.random_salt:/dim_stack/.random_salt\n    networks:\n      - dimstack-net\n\n# ==================== 网络配置 ====================\nnetworks:\n  dimstack-net:\n    driver: bridge\n```\n\n### 🔍 配置详解\n\n| 配置项 | 说明 |\n|--------|------|\n| `restart: unless-stopped` | 服务器重启后自动恢复容器 |\n| `127.0.0.1:3307:3306` | MySQL 仅监听本地，避免暴露到公网 |\n| `127.0.0.1:6380:6379` | Redis 仅监听本地，安全起见 |\n| `depends_on + service_healthy` | 等待数据库就绪后再启动应用 |\n| `dimstack-net` | 自定义桥接网络，容器间通过服务名通信 |\n| `healthcheck` | 健康检查确保服务真正可用 |\n\n---\n\n## 6. 构建与启动\n\n### 🏗️ 构建镜像\n\n```bash\ncd /opt/dim_stack\n\n# 首次构建（或 Dockerfile NPM 反向代理配置\n\n为了使用域名访问并通过 HTTPS 加密传输，我们使用 **Nginx Proxy Manager**（NPM）进行反向代理。\n\n### 🏗️ 架构说明\n\nNPM 是一个基于 Nginx 的可视化反向代理管理工具，支持通过 Web UI 配置代理规则和 SSL 证书。\n\n### 🔧 通过 NPM API 创建代理主机\n\n如果你已经部署了 NPM，可以通过 API 或 Web UI 创建代理规则：\n\n```bash\n# 通过 API 创建代理主机\ncurl -X POST "http://npm-host:81/api/nginx/proxy-hosts" \n  -H "Authorization: Bearer *** \n  -H "Content-Type: application/json" \n  -d '{\n    "domain_names": ["whalemalus.com"],\n    "forward_host": "172.18.0.1",\n    "forward_port": 2222,\n    "ssl_forced": true,\n    "hsts_enabled": true,\n    "http2_support": true,\n    "block_exploits": true\n  }'\n```\n\n### 📋 配置要点\n\n| 配置项 | 值 | 说明 |\n|--------|-----|------|\n| 域名 | `whalemalus.com` | 你的博客域名 |\n| 转发地址 | `172.18.0.1:2222` | Docker 宿主机在 NPM 网络中的 IP |\n| SSL | Cloudflare Origin Certificate | 源站证书 |\n| HSTS | 启用 | 强制 HTTPS |\n| HTTP/2 | 启用 | 提升性能 |\n\n### 🔐 SSL 证书配置\n\n1. 在 Cloudflare 控制面板生成 **Origin Certificate**\n2. 复制证书内容和私钥\n3. 在 NPM 的 SSL 选项卡中上传证书\n4. 启用「Force SSL」和「HTTP/2」\n\n### 📝 Web UI 操作步骤\n\n1. 登录 NPM 管理面板（通常在 `http://<IP>:81`）\n2. 进入 **Hosts** → **Proxy Hosts** → **Add Proxy Host**\n3. 填写域名：`whalemalus.com`\n4. 填写转发目标：`172.18.0.1`，端口：`2222`\n5. 切换到 **SSL** 选项卡：\n   - 选择「Custom」证书类型\n   - 粘贴 Origin Certificate 和 Private Key\n   - 勾选 Force SSL、HSTS、HTTP/2\n6. 保存\n\n---\n\n## 9. 最终架构总览\n\n### 🏛️ 请求流转路径\n\n```\n用户浏览器\n    │\n    ▼\n┌─────────────────────────────────┐\n│   Cloudflare CDN / DNS / SSL    │  ← 全球加速 + 边缘 SSL\n└─────────────┬───────────────────┘\n              │ HTTPS (443)\n              ▼\n┌─────────────────────────────────┐\n│   Nginx Proxy Manager (NPM)     │  ← 反向代理 + 源站 SSL 终结\n│   监听: 443                     │\n└─────────────┬───────────────────┘\n              │ HTTP (2222)\n              ▼\n┌─────────────────────────────────┐\n│   Dim Stack Application         │  ← Spring Boot 4 + React 19\n│   容器: dimstack-app            │\n│   端口: 2222                    │\n└──────┬──────────────┬───────────┘\n       │              │\n       ▼              ▼\n┌──────────────┐ ┌──────────────┐\n│ MySQL 8.0    │ │ Redis 7      │\n│ 容器:        │ │ 容器:        │\n│ dimstack-mysql│ │ dimstack-redis│\n│ 端口: 3306   │ │ 端口: 6379   │\n└──────────────┘ └──────────────┘\n```\n\n### 🔒 安全层级\n\n| 层级 | 安全措施 |\n|------|----------|\n| 第一层 | Cloudflare WAF + DDoS 防护 |\n| 第二层 | NPM 反向代理 + SSL/TLS |\n| 第三层 | Docker 网络隔离 |\n| 第四层 | MySQL/Redis 仅本地监听 |\n\n---\n\n## 10. 中使用了清华镜像源，海外服务器访问慢或不可达\n\n**解决**：\n\n```dockerfile\n# 注释掉中国镜像配置\n# RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.tuna.tsinghua.edu.cn/g' /etc/apk/repositories\n```\n\n---\n\n### ❓ 问题二：host.docker.internal 解析失败\n\n**症状**：应用无法连接 MySQL/Redis，报错 `UnknownHostException: host.docker.internal`\n\n**原因**：`host.docker.internal` 在 Linux 上默认不支持（仅 Docker Desktop 支持）\n\n**解决**：使用 Docker Compose 的自定义网络，通过**服务名**访问：\n\n```yaml\n# ❌ 错误写法\nspring.datasource.url=jdbc:mysql://host.docker.internal:3306/dim_stack\n\n# ✅ 正确写法（Docker 网络内）\nspring.datasource.url=jdbc:mysql://mysql:3306/dim_stack\n```\n\n---\n\n### ❓ 问题三：容器间网络不通\n\n**症状**：应用容器无法连接到 MySQL 或 Redis 容器\n\n**原因**：容器不在同一个 Docker 网络中\n\n**解决**：确保 `docker-compose.yml` 中所有服务都加入了同一个网络：\n\n```yaml\nnetworks:\n  dimstack-net:\n    driver: bridge\n```\n\n验证网络连通性：\n\n```bash\n# 从应用容器 ping MySQL\ndocker exec dimstack-app ping mysql\n\n# 检查网络\ndocker network inspect dimstack-net\n```\n\n---\n\n### ❓ 问题四：MySQL 初始化超时\n\n**症状**：应用启动时数据库连接失败，MySQL 容器日志显示仍在初始化\n\n**原因**：MySQL 首次启动需要较长时间初始化数据目录\n\n**解决**：已在 `docker-compose.yml` 中配置健康检查和 `start_period`：\n\n```yaml\nmysql:\n  healthcheck:\n    test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]\n    interval: 10s\n    timeout: 5s\n    retries: 10\n    start_period: 30s   # 给 MySQL 30 秒启动时间\n```\n\n应用通过 `depends_on` + `condition: service_healthy` 等待 MySQL 就绪后再启动。\n\n---\n\n### ❓ 问题五：上传文件权限问题\n\n**症状**：上传图片失败，日志显示 `Permission denied`\n\n**原因**：宿主机目录权限与容器内用户不匹配\n\n**解决**：\n\n```bash\n# 修复权限\nchown -R 1001:1001 /opt/dim_stack/upload\nchmod -R 755 /opt/dim_stack/upload\n```\n\n---\n\n### ❓ 问题六：端口被占用\n\n**症状**：启动时报错 `Bind for 0.0.0.0:2222 failed: port is already allocated`\n\n**原因**：端口 2222 已被其他进程占用\n\n**解决**：\n\n```bash\n# 查找占用端口的进程\nlsof -i :2222\n\n# 杀掉占用进程\nkill -9 <PID>\n\n# 或修改 docker-compose.yml 中的端口映射\nports:\n  - "3222:2222"  # 改用其他端口\n```\n\n---\n\n## 12. 总结\n\n### 📝 部署清单\n\n- [x] 准备服务器环境（Docker、Docker Compose）\n- [x] 创建目录结构并设置权限\n- [x] 克隆项目并构建 JAR 包\n- [x] 适配 Dockerfile（处理镜像源问题）\n- [x] 编写
