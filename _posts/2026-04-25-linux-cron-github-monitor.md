Cron 基础：什么是定时任务\n\n### 1.1 Cron 的工作原理\n\ncron 是一个守护进程（daemon），它在后台运行，每分钟检查一次是否有需要执行的任务。\n\n```\n┌─────────────────────────────────────────────┐\n│                cron daemon                   │\n│                                             │\n│  每分钟检查一次 ──→ 是否有匹配当前时间的任务？ │\n│                      │                      │\n│                 Yes  │  No                  │\n│                 ↓    │  ↓                   │\n│            执行任务   │  继续等待             │\n└─────────────────────────────────────────────┘\n```\n\n### 1.2 Cron 的两个核心概念\n\n| 概念 | 说明 | 文件位置 |\n|------|------|----------|\n| **crontab** | 定时任务配置文件，每个用户一份 | `/var/spool/cron/crontabs/` |\n| **cron 表达式** | 定义"什么时候执行"的时间规则 | 写在 crontab 里 |\n\n### 1.3 查看当前用户的定时任务\n\n```bash\n# 列出当前用户的所有定时任务\ncrontab -l\n\n# 列出指定用户的定时任务（需要 root）\ncrontab -l -u username\n```\n\n## 2. Cron 表达式语法详解\n\n### 2.1 基本格式\n\ncron 表达式由 **5 个时间字段** + **1 个命令** 组成：\n\n```\n┌───────────── 分钟 (0-59)\n│ ┌─────────── 小时 (0-23)\n│ │ ┌───────── 日期 (1-31)\n│ │ │ ┌─────── 月份 (1-12)\n│ │ │ │ ┌───── 星期 (0-7, 0和7都是周日)\n│ │ │ │ │\n* * * * * command_to_execute\n```\n\n### 2.2 特殊符号\n\n| 符号 | 含义 | 示例 |\n|------|------|------|\n| `*` | 任意值 | `* * * * *` = 每分钟 |\n| `,` | 列举多个值 | `1,15 * * * *` = 每小时的第1和第15分钟 |\n| `-` | 范围 | `9-17 * * * *` = 每天 9:00 到 17:00 |\n| `/` | 步长 | `*/5 * * * *` = 每5分钟 |\n| `0` | 特定值 | `0 9 * * *` = 每天早上9点整 |\n\n### 2.3 常用表达式速查\n\n```bash\n# 每分钟执行\n* * * * * /path/to/script.sh\n\n# 每5分钟执行\n*/5 * * * * /path/to/script.sh\n\n# 每小时执行\n0 * * * * /path/to/script.sh\n\n# 每天凌晨2点执行\n0 2 * * * /path/to/script.sh\n\n# 每天早上9点和下午6点执行\n0 9,18 * * * /path/to/script.sh\n\n# 每周一早上9点执行\n0 9 * * 1 /path/to/script.sh\n\n# 每月1号凌晨3点执行\n0 3 1 * * /path/to/script.sh\n\n# 工作日（周一到周五）早上9点执行\n0 9 * * 1-5 /path/to/script.sh\n\n# 每12小时执行一次\n0 */12 * * * /path/to/script.sh\n```\n\n### 2.4 图解示例\n\n```\n0 9 * * 1-5 /home/user/backup.sh\n│ │  │ │ │\n│ │  │ │ └── 周一到周五\n│ │  │ └──── 每月\n│ │  └────── 每天\n│ └───────── 早上9点\n└─────────── 整点（第0分钟）\n\n= 工作日每天早上9点整执行备份\n```\n\n## 3. 创建和管理定时任务\n\n### 3.1 编辑定时任务\n\n```bash\n# 交互式编辑（推荐）\ncrontab -e\n\n# 直接设置（会覆盖已有任务！慎用！）\necho "0 9 * * * /path/to/script.sh" | crontab -\n```\n\n### 3.2 从文件导入\n\n```bash\n# `/home/user/scripts/github-issue-monitor.sh`：\n\n```bash\n#!/bin/bash\n# GitHub Issue 回复监控脚本\n# 功能：检查指定 Issue 是否有新回复，有则输出通知内容\n\n# ============ 配置区 ============\nOWNER="lingview"          # 信息和评论\nCOMMENTS=$(gh issue view $ISSUE_NUMBER \\\\\n  --repo "$OWNER/$REPO" \\\\\n  --comments 2>&1)\n\nISSUE_STATE=$(gh issue view $ISSUE_NUMBER \\\\\n  --repo "$OWNER/$REPO" \\\\\n  --json state -q '.state' 2>&1)\n\n# 检查是否成功获取\nif [ $? -ne 0 ]; then\n  echo "❌ 获取 Issue 失败: $COMMENTS"\n  exit 1\nfi\n\n# 计算当前评论数（简单方式：统计评论中的时间戳）\nCURRENT_COUNT=$(echo "$COMMENTS" | grep -c "^•")\n\n# 读取上次记录的评论数\nLAST_COUNT=0\nif [ -f "$STATE_FILE" ]; then\n  LAST_COUNT=$(cat "$STATE_FILE")\nfi\n\n# 比较\nif [ "$CURRENT_COUNT" -gt "$LAST_COUNT" ]; then\n  NEW_COUNT=$((CURRENT_COUNT - LAST_COUNT))\n  echo "🔔 GitHub Issue #${ISSUE_NUMBER} 有 ${NEW_COUNT} 条新回复！"\n  echo ""\n  echo "Issue: https://github.com/${OWNER}/${REPO}/issues/${ISSUE_NUMBER}"\n  echo "状态: $ISSUE_STATE"\n  echo ""\n  echo "最新评论:"\n  echo "$COMMENTS" | tail -20\n  echo ""\n  \n  # 更新状态文件\n  echo "$CURRENT_COUNT" > "$STATE_FILE"\nelif [ "$ISSUE_STATE" = "CLOSED" ]; then\n  echo "✅ GitHub Issue #${ISSUE_NUMBER} 已关闭"\n  echo "https://github.com/${OWNER}/${REPO}/issues/${ISSUE_NUMBER}"\n  \n  # 更新状态\n  echo "CLOSED" > "$STATE_FILE.closed"\nelse\n  echo "⏳ GitHub Issue #${ISSUE_NUMBER} 暂无新回复 (评论数: $CURRENT_COUNT)"\nfi\n```\n\n设置执行权限：\n\n```bash\nchmod +x /home/user/scripts/github-issue-monitor.sh\n```\n\n### 4.4 设置定时任务\n\n```bash\n# 编辑 crontab\ncrontab -e\n\n# 添加以下内容：\n# 每6小时检查一次 GitHub Issue 坑1：环境变量缺失\n\n**现象**：脚本手动运行正常，但 cron 执行时找不到命令\n\n**原因**：cron 的 PATH 环境变量很精简，不包含 `/usr/local/bin` 等路径\n\n**解决**：\n```bash\n# 在 crontab 开头设置 PATH\nPATH=/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin\n\n# 或者在脚本中设置\nexport PATH="/usr/local/bin:$PATH"\n```\n\n### 坑2：相对路径失效\n\n**现象**：脚本中使用了 `./config.yaml`，cron 执行时找不到文件\n\n**原因**：cron 的工作目录不是脚本所在目录\n\n**解决**：\n```bash\n# 使用绝对路径\nCONFIG="/home/user/scripts/config.yaml"\n\n# 或者在脚本开头切换目录\ncd "$(dirname "$0")"\n```\n\n### 坑3：输出没有重定向\n\n**现象**：cron 执行了但看不到结果，也没有日志\n\n**原因**：cron 的 stdout/stderr 默认发邮件，但系统没配邮件\n\n**解决**：\n```bash\n# 重定向输出到日志文件\n0 */6 * * * /path/to/script.sh >> /var/log/myscript.log 2>&1\n```\n\n### 坑4：权限不足\n\n**现象**：脚本手动运行正常，cron 执行时报 Permission denied\n\n**原因**：cron 以不同用户身份执行，权限可能不同\n\n**解决**：\n```bash\n# 确保脚本有执行权限\nchmod +x /path/to/script.sh\n\n# 如果需要 root 权限，使用 root 的 crontab\nsudo crontab -e\n```\n\n### 坑5：时区问题\n\n**现象**：cron 没有在预期时间执行\n\n**原因**：系统时区和预期不一致\n\n**解决**：\n```bash\n# 检查系统时区\ntimedatectl\n\n# 设置时区\nsudo timedatectl set-timezone Asia/Shanghai\n```\n\n## 7. 最佳实践\n\n### 7.1 命名规范\n\n```bash\n# 好的命名：描述性 + $LOCKFILE" EXIT\ntouch "$LOCKFILE"\n\n# ... 执行任务 ...\n```\n\n## 8. 总结\n\n### 核心要点\n\n| 要点 | 说明 |\n|------|------|\n| cron 表达式 | 5个字段：分 时 日 月 周 |\n| 环境变量 | cron 的 PATH 很精简，用绝对路径 |\n| 输出重定向 |
